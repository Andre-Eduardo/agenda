# Project Overview — Automo / Agenda

Healthcare/agenda management application. **Monorepo** managed with **pnpm workspaces**.

## Purpose
Full-stack platform for clinic and professional scheduling: appointments, patients, clinical documents, billing, forms, notifications, AI agents.

## Tech Stack
| Layer | Technology |
|-------|-----------|
| Backend | NestJS 11 + Prisma 6 + PostgreSQL |
| Frontend | React 19 + TanStack Router v1 + React Query v5 + Zustand v5 |
| UI | Tailwind CSS v4 + shadcn/ui (Radix UI) |
| Forms | React Hook Form + Zod |
| i18n | i18next (pt-BR default, en-US, es-ES) |
| Tests | Jest (unit) + Cucumber.js BDD (integration) |
| Linter | oxlint + oxfmt (NOT eslint/prettier) |
| Package manager | pnpm >= 9 |
| TypeScript | 5.8 (server), 6 (web) |

## Monorepo Structure
```
apps/
  server/   — NestJS backend (Node >= 18, TypeScript 5.8)
              src/domain/        — Domain layer (DDD aggregates per module)
              src/application/   — Application layer (use cases, commands, services)
              src/infrastructure/ — Infrastructure (DB, auth, storage, notifications, etc.)
  web/       — React 19 frontend (TypeScript 6, Vite 6)
              src/app/           — Route/feature pages
              src/components/    — Shared UI components
              src/hooks/         — Custom hooks
              src/store/         — Zustand stores
              src/views/         — View components
              src/translations/  — i18n locale files
packages/
  client/         — API client generated by Orval (React Query + Axios)
  value-objects/  — Shared immutable value objects
docs/             — Architecture and pattern documentation
```

## Architecture
- Backend follows **Clean Architecture + DDD**: domain → application → infrastructure layers
- Multi-tenant isolation: `professionalId` cookie → Actor → mandatory filter in all repositories
- Frontend uses generated API client (never write API calls manually)
